# 用語集（Glossary）

英文ラボ（Eibun-Lab）で使う日本語の用語は、このファイルを**唯一の正典**とする。
UI 文言・Gemini プロンプト・ドキュメントのいずれを書くときも、ここの「正表記」に従う。

このファイルに対応する機械可読ルールが `prh.yml` にあり、`npm run lint:text` で自動チェックできる。
**新しいドメイン用語を導入したら、この表と `prh.yml` の両方に追記すること。**

---

## なぜ用語集が必要か

<!-- textlint-disable -->

同じ概念が「添削」と「校正」の2通りで書かれていると、次の3つが同時に壊れる。
<!-- textlint-enable -->

1. **ユーザーの理解** — 画面ごとに呼び名が変わると、別機能だと誤解される。
2. **Gemini の出力** — `prompt.util.ts` の見出しがそのまま AI の出力語彙を規定するため、
   プロンプトの用語がぶれるとアプリ側の表示語彙と乖離する。
3. **コードとの対応** — 「セッション」は型 `CorrectionSession` に対応する。
   日本語表記がぶれると、UI 文言からコードを辿れなくなる。

## 「ミス」と「誤り」「間違い」の使い分け

**「ミス」への統一は、UI ラベルと見出しにのみ適用する。**
散文・コメント・プロンプトの説明文における「誤り」「間違いやすい」は自然な日本語であり、**禁止しない**。

許容される例（いずれも修正不要）:

- `src/app/core/gemini/prompt.util.ts:31` — 「文法・語法の**誤り**を指摘し」
  （「文法・語法のミスを指摘し」より自然。動詞と結びつく散文中の用法）
- `src/app/core/gemini/prompt.util.ts:58` — 「**間違いやすい**癖」
  （「ミスやすい」は非文。形容詞的用法は言い換え不能）
- `src/app/features/drill/drill.ts:268` — 「押し**間違い**」
  （複合名詞。「押しミス」は不自然）

統一すべき例（`prh.yml` が検出する）:

<!-- textlint-disable -->

- 「間違い一覧」→ **ミス一覧**（`prompt.util.ts` のセクション見出し）
- 「間違い傾向」→ **ミス傾向**（`features/mistakes` の画面名）

<!-- textlint-enable -->

この方針のため、`prh.yml` には裸の「間違い」「誤り」をパターンとして登録していない。
登録すると上記の自然な日本語が大量に誤検知され、ルール自体が使われなくなるためである。

## チェック方法

```bash
npm run lint:text      # 違反を検出する
npm run lint:text:fix  # 検出した違反を正表記に自動置換する
```

対象は `docs/` 配下の Markdown、`README.md`、`src/app/core/gemini/prompt.util.ts`。
`prompt.util.ts` は Gemini の出力語彙を直接規定するため、ソースコードの中で唯一チェック対象に含めている。
